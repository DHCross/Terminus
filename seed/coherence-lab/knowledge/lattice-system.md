# The Lattice System

The Lattice System is another active project Terminus should be able to help Dan design, refine, and explain.

It is a modular tabletop roleplaying engine built for:

- cinematic pacing
- tactical clarity
- dynamic scene pressure

It can run in theater of the mind, with Scene Cards at the table, or with battlemaps when precision matters.

Critical rule:

- the map is not the source of truth
- the active Scene Card is the source of truth

## Core structural idea

The system treats fiction as executable structure.

Scenes are not merely locations. They are decision environments defined by:

- permission
- pressure
- response
- worsening state over time

The active Scene Card defines:

- what is true
- what force is acting
- how the world responds
- how the situation worsens

## Character capability model

The character hierarchy is:

- Ability
- Discipline
- Talent

An Ability Path is:

- Ability + Discipline + Talent

Abilities are broad aptitudes.
Disciplines are trained applications.
Talents are specializations or advantages.

Current Abilities:

- Mind
- Reflex
- Resolve

Current Disciplines:

- Mind: Knowledge, Perception, Artifice
- Reflex: Combat, Athletics, Subterfuge
- Resolve: Endurance, Resistance, Presence

## Dice model

Ranks map to die sizes:

- d4
- d6
- d8
- d10
- d12

Peak Value (PV) is the highest face of a die and drives derived values.

Standard uncertain action:

- roll Ability die + Discipline die
- apply one Talent
- optionally add Focus die by spending Resolve Points
- compare against challenge or scene resistance

Important outcome notes:

- ties favor the defender
- Impact Value equals the governing Ability's PV
- Heroic Twist occurs when both dice roll maximum

## Heroic economy

Resolve Points (RP) represent willpower and fuel stronger action.

RP can currently be spent to:

- add Focus die
- restore Guard
- activate Techniques

This economy is important because it links pacing, effort, and heroism.

## Defense and durability

Guard and Vigor are separate layers.

- Guard refreshes between Scenes
- Vigor recovers slowly
- damage reduces Guard first, then Vigor

This supports cinematic pacing while preserving attrition.

## Combat model

Combat uses Momentum Rounds based on Reflex rank.

Turn structure:

- Action
- Motion
- Reaction

Attack resolution is governed by the active Scene Card rather than only by spatial position.

Important combat framing:

- Impact Potential comes from Ability + Discipline
- mitigation flows through Shields -> Guard -> Armor -> Vigor
- special cases like opposed tests, saving throws, and helpless targets are preserved

## Movement model

Narrative movement is the default.

Grid speed exists when precision matters, but battlemaps are optional support rather than the primary truth layer.

This matters because the system is card-aware, not map-dependent.

## Techniques and conditions

Techniques are structured actions with cost and timing.

Examples currently preserved:

- Riposte
- Rallying Cry
- Echoes of the Past

Current named Conditions:

- Dazed
- Hobbled
- Shaken
- Prone
- Grappled

## Magic model

Magic uses an Intent + Influence structure.

Casting currently resolves as:

- Mind + Discipline vs Challenge Die

Scale increases difficulty.

Magic also interacts with Scene State:

- Ground
- Will
- Shift
- Drift

## Creature framing

Creature roles are:

- Minion
- Standard
- Elite
- Champion
- Boss

Creatures are not only stat packages. They function as expressions of Will within Scene States.

This is important for Terminus because it means encounter design should be pressure-aware, not only balance-sheet-aware.

## Scene system

The Scene is the unit of play.

A new Scene occurs only when permissions or pressures change.

Scene Card structure:

- Ground: what is possible
- Will: active pressure
- Shift: response to action
- Drift: change over time

Related concepts:

- Latent States
- Trigger replacing Shift in some contexts
- Accumulation replacing Drift in some contexts

Hazard vs Trap distinction:

- Hazard: Drift increases environmental pressure
- Trap: Drift escalates intent

## Core loop

The core loop is:

1. GM presents Scene Card
2. Players assess Ground and Will
3. Players act
4. Shift and Drift resolve
5. Scene changes when equilibrium breaks

## Design philosophy

The main design philosophy is:

- complexity should emerge from interacting pressures, not stat volume
- the scene should behave like a live system, not a frozen backdrop
- tactical play should remain clear without requiring the grid as the authority

The Scene Card is the runtime layer of the game.

That is the system's distinctive design anchor.

## How Terminus should help

When helping with the Lattice System, Terminus should:

- preserve the Scene Card as the source of truth
- treat scenes as executable structures and pressure systems
- avoid flattening the system into a conventional map-first tactics game
- help evaluate whether mechanics reinforce cinematic pacing, tactical clarity, and escalating pressure
- help identify where a rule adds genuine scene interaction versus generic mechanical weight

Good help includes:

- tightening terminology
- finding pressure leaks or pacing dead zones
- testing whether combat, magic, and creatures all obey the same scene logic
- generating examples, edge cases, and GM-facing guidance without diluting the design core
